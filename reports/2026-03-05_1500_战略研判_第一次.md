## 二、研究员结论的深度审查

### 结论 1: "Per-class balanced loss 解决了类别竞争震荡"
**研判: 部分成立, 但过于乐观。**

证据支持:
- Plan B iter 5000 首次实现 4 类同时有意义的 recall (car=0.757, truck=0.501, bus=0.597, trailer=0.844)
- Baseline 同期 (iter 2000) truck 崩溃到 0.002, Plan B 最低 0.06

证据反驳:
- iter 2500 truck 仍从 0.35 跌到 0.06 (跌幅 83%), 这不是"解决"而是"缓解"
- iter 5000 后所有指标持续下滑 (avg_R: 0.675 -> 0.533), 说明 LR decay 后模型失去了学习能力
- 研究员在 Check #17 就声称 "OSCILLATION BROKEN" (仅 3 个 checkpoint), 监督员已指出这一判断过早, 后续果然被打脸

**我的判定: Per-class balanced loss 是正确方向, 将震荡幅度从 [0.002, 0.65] 缩小到 [0.06, 0.50], 但并未根本解决。振荡的根源可能在 target assignment 层面, 不仅仅是 loss aggregation。**

### 结论 2: "精度低是训练端问题, 推理后处理无法解决"
**研判: 基本成立, 但推理有重大漏洞。**

关键发现链:
1. Score filtering 从未生效 (`p_scores` 一直是 None) -- 这是代码 bug, 不是算法问题
2. Marker softmax 均值 0.97 -- 模型对所有预测过度自信
3. NMS 有害 (recall 0.69 -> 0.025)

**我的质疑:**

**(a) Score filtering 从未生效, 那 iter 3500-10000 的 "新评估器" 结果其实等同于旧评估器。** 研究员在 Check #19 做了精度根因分析, 然后部署了修复, 但修复实际上没生效。这意味着:
- iter 3500-10000 的数据是可信的 (因为没有被错误的 filter 污染)
- 但研究员在 report 中写 "FIRST with score filtering" 是错误的标注

**(b) Marker softmax = 0.97 并不意味着训练不能解决精度问题。** 它意味着当前的 loss 函数没有对 confidence calibration 施加压力。如果加入 calibration loss (如 label smoothing, temperature scaling, 或 explicit confidence loss), 模型完全有可能学会区分 TP 和 FP。

**(c) NMS recall 暴跌到 0.025 说明什么?** 说明 NMS 的 IoU 阈值设置可能有问题, 或者 BEV 空间的 box 表示不适合标准 NMS。这不等于 NMS 不可行, 只是需要调参。研究员过早否定了 NMS。

**(d) "~24.5% 空槽误报" 是核心问题。** 每张图 ~257 个假阳性 vs ~26 个真阳性 (以 car 为例)。这不是后处理能解决的, 研究员判断正确。但 24.5% 的 bg_false_alarm 在 10000 iter 的训练后仍然这么高, 说明模型对 "空 vs 非空" 的判别能力学习不足。

### 结论 3: 最佳 checkpoint 是 iter 5000
**研判: 成立, 但有条件。**

iter 5000 确实是 avg_recall 最高的点 (0.675), 且 LR 衰减后模型退化。但:
- 这个 "最佳" 是在 per-class balanced loss 这一特定设计下的最佳
- 如果 loss 设计改变 (比如加入 confidence loss, 或更激进的 bg 惩罚), 可能有完全不同的训练曲线
- iter 2000 的 Plan B checkpoint (avg_R=0.71, 4 类同时活跃) 可能是更好的 fine-tuning 起点

### 如果 ALLOW_COMMANDS 改为 True, 我的指令将是:

**优先级 1 -- 修复 score 传播 bug:**
研究员已定位问题 (`p_scores` 为 None), 但尚未修复。这是所有后续精度优化的前提。修复后在 iter 5000 checkpoint 上验证 score distribution, 确认 threshold filtering 确实生效。

**优先级 2 -- 精度改善的训练策略 (二选一):**

方案 A: **Background confidence loss**
- 在现有 per-class balanced loss 基础上, 加入对空槽预测的 confidence penalty
- 具体: 对 marker 预测为 END 的 loss 加权, 或对非 END 预测加 entropy regularization
- 从 iter 2000 checkpoint 开始 (最平衡的起点)

方案 B: **Hard negative mining**
- 每 batch 只对 bg_false_alarm 中 confidence 最高的 top-K 个空槽反传梯度
- 这是 object detection 领域成熟的方案 (OHAI, OHEM)
- 同样从 iter 2000 checkpoint 开始

**我倾向方案 A**, 因为它更简单, 与 per-class balanced loss 兼容, 且不需要额外的 mining 逻辑。

**优先级 3 -- NMS 不应被放弃:**
研究员用默认参数测试 NMS 一次就否定了它。BEV 空间的 rotated NMS 需要仔细调参 (IoU threshold, class-aware NMS)。但这是优先级 3, 先解决训练端精度再考虑。
