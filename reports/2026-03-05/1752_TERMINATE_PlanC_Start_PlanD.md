## 三、Plan C 死因分析

**表面现象**: 类振荡 + precision 全面崩塌

**根因**: reg_loss_weight=2.0 导致回归梯度压制分类梯度

证据链：
1. **bg_FA 轨迹**: 0.156 -> 0.171 -> 0.206 -> **0.294** (单调上升)
   - bg_balance_weight=3.0 在 iter 500-1000 有效，但被 reg 梯度逐步稀释
   - 到 iter 2000 bg_FA 已经超过 Plan B 水平 (0.253@1000)

2. **Precision 全线崩塌**:
   - car_P: 0.143 -> 0.104 -> 0.074 -> 0.081 (最终比 Plan B 还差)
   - bus_P: 0.113 -> 0.229 -> 0.115 -> 0.062 (iter 1000 的达标只是昙花)
   - trailer_P: 0.013 -> 0.061 -> 0.020 -> 0.007 (几乎为零)

3. **类振荡模式** (典型的梯度竞争):
   - iter 1000: bus 上升 + truck/trailer 下降
   - iter 1500: truck/trailer 继续下降, bus 也开始下降
   - iter 2000: bus/truck/trailer 反弹, 但 car 被挤压 + bg_FA 爆表

4. **car_R 单调下降**: 0.697 -> 0.598 -> 0.542
   - car 是最大类 (8267 GT)，不应该持续下降
   - 说明分类能力在整体退化，不只是类间竞争

**结论**: bg_fix (P0 修复) 的方向正确，但 reg_loss_weight=2.0 产生的梯度远大于 cls，长期稀释了 bg_fix 的效果。模型在"学习更好的空间位置"和"学习更好的检测"之间被迫选择了前者。

## 四、Plan D 设计

| 参数 | Plan C | Plan D | 理由 |
|------|--------|--------|------|
| reg_loss_weight | 2.0 | **1.0** | 恢复 cls/reg 梯度平衡 |
| bg_balance_weight | 3.0 | 3.0 | 在 iter 500-1000 证明有效 |
| load_from | baseline@2000 | **Plan C@1000** | C@1000 是最佳检查点 |
| max_iters | 8000 | 6000 | 从更好的起点出发 |

**为什么从 C@1000 而非 C@500**:
- C@1000 是唯一有两个类别达标的检查点 (bus_R=0.609, bus_P=0.229)
- bg_FA=0.171 仍在可控范围
- C@500 的 bg_FA=0.156 更低，但类别学习不够成熟

**预期效果**:
- reg_weight=1.0 使 cls 梯度占比从 ~25% 提升到 ~50%
- bg_fix 不再被 reg 梯度稀释，bg_FA 应保持 < 0.20
- Precision 应有实质性提升（更多梯度用于分类而非定位）
- 代价：空间精度可能略有下降 (offset 指标可能上升)
